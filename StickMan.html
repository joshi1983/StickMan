property AutoUpdate: Boolean

AutoUpdate determines if the stickman should be updated when the stickman is drawn. Initially, this value is true.
When you change this property, the AutoUpdate property of the stickman's arms, legs, and eyes also are set to the same value.

property Bearded: Boolean

Determines if the stickman has a beard

property Canvas: tCanvas

The canvas, the stickman is drawn on
See UseDoubleBuffer for suggestions about certain canvases.

procedure DrawStickMan;

Draws the stickman on his canvas

property EyeColour: tColor;

This property can be used to set the colour of the stickman's eyes.
You can also use LeftEye.EyeColour and RightEye.EyeColour but since each eye usually has the same colour, this property is more efficient.

property EyebrowStyle: byte;

This property determines the type of eyebrow the stickman has.
Same values include ebBushy, ebReallyBushy, ebLifted, ebVeryThin, ebArced.

property HairColour: tColor

The colour of the stickman's hair and his beard, if he has one

property HairStyle: tHairStyle

The stickman's style of hair

Posible Values
hsBald Doesn't show any hair
hsNearBald The stickman has lost a lot of hair but he still has some on the sides
hsNormal The guy doesn't have any particullar hair style. This is the initial hair style.
hsThick The stickman has big fluffy hair like Richard Simmons.
hsMohawk The stickman has a little patch on the top of his head.
hsLongStraight The stickman has long straight hair.

property HeadPos: tPoint

The point in the centre of of the stickman's head.

This is the height of the stickman. Other features are scaled to the height of the stickman.

procedure RemoveObject

Get the stickman to remove an object.
This procedure starts an animation that ends with the stickman destroying the object.
The object can't be of any type. It must be a tButton, tRadioGroup, tCheckBox, tLabel, or tEdit.

property ReplaceBackgrounds: boolean

Determines if the previous drawing of the stickman should be replaced by a stored bitmap of the region before any of the stickman was drawn
Initially, this value is true.
When this property is true, the background before each drawing of the stickman is stored. Also, before each drawing of the stickman, the background is replaced with this stored bitmap. This is not to be confused with the doublebuffer, though.
If you have a background that changes(ie is animated), I suggest setting this property to false. If you leave it true, within the stored rectangle of the stickman(the part replaced), your animation may be flickery and dissappear.

procedure Update

Update the stickman's animation
This also updates the stickman's arms, legs and eyes.
This procedure is automatically called when you draw the stickman unless you set AutoUpdate to false.

property UseDoubleBuffer: Boolean

This is initially true.
The double buffer is used to prevent flickery graphics. It ensures that when the stickman is drawn, it is drawn on a second canvas before being copied to the stickman's canvas. This is good for when the stickman is on a form's canvas. If the stickman is on a bitmap's, an image, or any canvas that won't show the flicker problem, I recommend setting UseDoubleBuffer to false. This will save memory and make the graphics a little faster to draw.
